## Anti-Patterns Verdict

**Тело протокола чистое** — телов сырого AI-вывода нет: русские заголовки, эмодзи по словарю, филлеры гасятся системно, пустые секции скрыты, экранирование полное. **Рамка машинная**: сводка с «🤖 ИИ: model-id, 84213 символов, сжатие 37%», fallback-транскрипция под видом протокола, «SOROKA» в колонтитуле PDF, имя файла от исходной записи (voice_message_123.pdf).

**Deterministic scan**: detect.mjs — 0 срабатываний (шаблоны + protocol_render), ложных нет. **Browser overlays**: пропущено — веб-поверхности нет.

## Overall Impression

Редизайн сработал: канальный слой соответствует ADR, «ничего пустого» защищено дважды, контракт доставки честный. Главный разрыв теперь не в теле протокола, а вокруг него: сводка на legacy-разметке прямо над идеальным телом, недостижимый флагманский шаблон ОД и молчаливые деградации.

## What's Working

1. Канальный слой по ADR: canonical Markdown → HTML с экранированием → посекционный сплит с «(продолжение)».
2. «Ничего пустого» дважды: {% if %} + нормализация филлеров до рендера.
3. Честный контракт доставки: статус по телу, предупреждение при потере, PDF→md деградация.

## Priority Issues

1. **[P1] «Протокол ОД» недостижим.** is_default=False исключает его из списка выбора (template_repo.py:31) И из автоподбора; категория management мертва — её нет в _CATEGORY_KEYWORDS (smart_template_selector.py:14–73). Флагманский сценарий PRODUCT.md — фикция. Фикс: точка входа (категория/кнопка) или честное удаление. → /impeccable onboard или harden
2. **[P1] Пустой протокол уходит молча.** ОД без поручений: шапка проходит порог >50 символов и уходит без секций (protocol_formatter.py:177). Фикс: else-ветка «Поручений не зафиксировано» или проверка содержательности перед отправкой. → /impeccable harden
3. **[P1] Fallback выдаёт сырую транскрипцию под видом протокола.** Три пути к «# Протокол\n\n{transcription}» без предупреждения (protocol_formatter.py:83, 200, 434). Фикс: префикс «⚠️ Не удалось структурировать…» + отдельный статус. → /impeccable harden
4. **[P1] Сводное сообщение — legacy Markdown.** **жирный** теряется, escape_markdown даёт видимые бэкслеши (message_builder.py:176–290, result_sender.py:57–58). Фикс: перевести сводку на HTML-рендер и сократить до 3–4 строк. → /impeccable clarify + distill
5. **[P2] Сплиттер: нет «часть N/M», _hard_wrap может разорвать парный тег.** Разбалансировка <b> на строке >4000 → parse-фейл → сырые теги у пользователя (splitter.py:71–85, telegram_rate_limiter.py:280–305). Фикс: балансировка тегов на границе + нумерация частей. → /impeccable harden

## Persona Red Flags

- **Alex (эксперт, ОД)**: не может выбрать спец-шаблон ни одним способом; «переделать» = заново грузить часовую запись.
- **Casey (мобильный)**: 4 части без нумерации — не видно, всё ли дошло; режим «файл .md» на телефоне — plain text с решётками.
- **Руководитель с форвардом**: «часть 2» без названия встречи и даты; PDF с «SOROKA» на каждой странице и именем файла от записи.

## Minor Observations

- «## Сроки» дублирует «## 📌 Задачи и сроки» в Стандартном (template_library.py:64–71).
- ADR ≠ код в деталях: «Лекция» без ✅/📌; «Повестка» до Решений в двух шаблонах.
- «Вчера»/«Сегодня» в Дейли — главный контент — после общих секций; предсказуемо, но спорно по релевантности.
- *курсив* в чате остаётся звёздочками, в PDF конвертируется — расхождение каналов; `код` в PDF — буквальные бэктики.
- Эмодзи в содержимом (не заголовках) → квадраты в PDF.
- _FILLER_RE не знает «неизвестно», «не определено», «нет».
- format_speaker_mapping_message — похоже, мёртвый код (вызовов не найдено).
- Справка по шаблонам обещает переменные, которых нет в системных шаблонах (message_builder.py:344–376).
- tempfile.mktemp — deprecated/небезопасен (result_sender.py:87, 94).

## Questions to Consider

1. Кто хоть раз получил «Протокол ОД», если его физически нельзя выбрать — не фикция ли главный сценарий продукта?
2. Почему над протоколом, который «пересылается без правок», бот кладёт сообщение с model-id и процентом сжатия — то, что перешлют вместе с ним?
