## Anti-Patterns Verdict

**Сам протокол чист** — теллов нет: русские заголовки, навигационные эмодзи, честные деградации, филлеры вычищаются. **Slop переехал в сервисный слой**: welcome/help с эмодзи на каждой строке и «живеньким» тоном против собственных анти-референсов; ~130 отправок `**жирный**` с legacy parse_mode="Markdown" (жирный теряется, при parse-fail — сырые звёздочки); мёртвый код как артефакт (переписанная справка недостижима — подтверждено: ноль вызовов; JSON-защита не подключена к пайплайну).

**Deterministic scan**: detect.mjs — 0 срабатываний (шаблоны, protocol_render, message_builder). **Browser**: неприменимо.

## Overall Impression

Ядро дожато: канальный рендер аккуратен (экранирование, баланс тегов, «(продолжение)», parse-fail деградация), пустота честная, сводка и имя файла сделаны для пересылки. Оценка не выросла потому, что ревью сняло следующий слой: сервисная обвязка (welcome, прогресс, списки шаблонов) говорит битой legacy-разметкой, части многочастного протокола не самодостаточны, а справка — мёртвый код. Главный сценарий «пересылка наверх» верен для целого протокола, но не для его части.

## What's Working

1. Канальный рендер по ADR-0001: экранирование, «(продолжение)», баланс тегов, hard-wrap без разрыва сущностей, деградация при parse-fail.
2. Честная пустота: филлеры → '', пустые секции исчезают, пустой ОД говорит явно, fallback-расшифровка помечена.
3. Сводка и имя файла для пересылки: техдетали в логах, файл называется по встрече.

## Priority Issues

1. **[P1] Части 2..M не самодостаточны.** Пересланная «Часть 2/3» — документ без названия встречи и даты; главный сценарий продукта ломается на самом частом действии. Фикс: повторять title-строку в начале каждой части (splitter.py:140–175). → /impeccable harden
2. **[P1] Сервисный слой говорит сломанной разметкой.** ~130 отправок `**bold**` с legacy Markdown: welcome/help, прогресс-трекер, очередь, флоу шаблонов. ADR закрыл тело протокола — обвязку нет. Фикс: перевести на HTML (message_builder.py:37–88, progress_tracker.py, queue_tracker.py, template_handlers.py, template_callbacks.py). → /impeccable harden + quieter (welcome-тон)
3. **[P2] Справка врёт, настоящая мертва.** /help предлагает удалённый шаг «Выберите ИИ» и несуществующие шаблоны; templates_help_message (единственное место с {% if %}) не привязана к хендлеру — подтверждено. Фикс: обновить /help, подключить справку к /templates. → /impeccable clarify
4. **[P2] Кастомные шаблоны молча нарушают «ничего пустого».** Флоу создания не упоминает {% if %}, валидация ловит только синтаксис. Фикс: пример с условиями в флоу + предупреждение при сохранении без {% if %}. → /impeccable onboard/harden
5. **[P2] Качели статусов при пустом протоколе.** Сводка уходит до проверки protocol_text (result_sender.py:202→204) — подтверждено. Фикс: проверку раньше сводки. → /impeccable harden

## Minor Observations

- meeting_title без правила в FIELD_SPECIFIC_RULES — качество заголовка (и имени файла) не управляется; переменные уходят в промпт с пустыми описаниями.
- JSON-защита (fix_json_in_text/convert_complex_to_markdown) не вызывается из пайплайна — dict/list от LLM в fallback-путях уйдёт как Python-repr.
- Эмодзи-диапазон PDF не покрывает U+2000–25FF (⏱ ▶ ‼ — тофу); #### — буквальный текст; `код` в PDF — Courier без кириллицы.
- ⭐ есть в одних списках шаблонов, нет в других; системные шаблоны без category — категории /templates пусты, ОД не найти под «Управленческие».
- Таксономия типов встреч LLM не знает «management» — типоспецифичных инструкций для главного руководительского сценария нет.
- «📅 Следующая встреча» (Дейли) наполняется next_steps («следующие шаги») — заголовок и контент могут разъехаться.
- Протокол из одних «---» → [] сообщений и delivered=True — ложный успех (экзотика).
- «⏱ Время: 312 с» — для долгих обработок мм:сс читабельнее.
- Резерв балансировки 16 < худшего случая 22 — спасает случайный двойной запас PART_PREFIX_RESERVE.
