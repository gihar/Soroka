## Anti-Patterns Verdict

**Сам протокол чист — проверено на живом рендере**: без английских заголовков, JSON-огрызков, служебных подписей и филлеров; эмодзи — только санкционированные метки. **Slop остался в chrome**: тексты установки/сброса шаблона по умолчанию — «живенький SMM-тон» из анти-референсов (стены текста, ✅🤖📊💡, «рекомендуемый режим для большинства пользователей»); англицизм «📁 Educational» нарушает «Один язык».

**Deterministic scan**: detect.mjs — 0 срабатываний. **Browser**: неприменимо.

## Overall Impression

Плато 25 пробито: 28/40, впервые с v1 балл вырос. Документ-протокол теперь оценивается на 30+ — граница Markdown→HTML, «ничего пустого» на трёх слоях, честная деградация и бриф как единый источник структуры признаны сильными сторонами свежим ревью без знания истории. P0 и P1 закончились — впервые весь список проблем состоит из P2/P3, и все они в chrome (подписи категорий, тон меню, пикер), а не в артефакте для форварда. Два содержательных вопроса остались: консистентность перегенерации (v2 может назвать участников иначе, чем ушедший наверх v1) и порядок секций Стандартного (повестка перед решениями против принципа №5).

## What's Working

1. **Единая граница Markdown→HTML** (render_legacy_markdown в rate limiter): класс багов «видны ## и **» закрыт разом для всего продукта.
2. **«Ничего пустого» на нескольких слоях**: {% if %} в брифах + нормализация филлеров + empty_text ОД.
3. **Честная деградация**: fallback подписывает себя, «Участник N» вместо меток, пустой протокол — внятный ретрай.
4. **Бриф — единый источник**: схема, правила промпта и контент из одной декларации; ключи не могут разъехаться.

## Priority Issues

1. **[P2] Тройной рассинхрон подписей категории + англицизм.** educational: «🎓 Образовательные» (mgmt:39) / «📚 Учебные» (back_to_templates:200) / «📁 Educational» (фолбэк .title() в command_handlers:123 и mgmt:475 — ключа educational там нет). Три имени одной сущности, одно не на том языке. Фикс: единый CATEGORY_LABELS-словарь, убрать .title()-фолбэк. → /impeccable distill
2. **[P2] Перегенерация заново «слушает» запись.** regenerate_protocol идёт с meeting_type=None, история не хранит speaker_mapping/meeting_type — ЭТАП 1 прогоняется с нуля: v2 может назвать участников иначе, чем ушедший форвардом v1, и сменить тип встречи; плюс лишний LLM-вызов. Фикс: сохранять speaker_mapping/meeting_type в processing_history и переиспользовать. → /impeccable harden
3. **[P3] Пикер перегенерации — плоский список во весь экран.** 10–15 кнопок в один столбец без категорий и отмены; рядом пикер шаблона по умолчанию — 2 колонки. Фикс: переиспользовать единый пикер. → /impeccable distill
4. **[P3] Chrome-тон off-brand.** set/reset default template — стены текста с маркетингом против «лаконичный/нейтральный». Фикс: 1–2 строки статуса. → /impeccable quieter
5. **[P3] Справка учит анти-паттерну.** Пример в templates_help: «Дата: {{ date }}» без {% if %} — оставит «Дата: » над пустотой, о чём справка предупреждает абзацем выше; стиль расходится с системной шапкой. Фикс: обернуть в {% if %}, привести к стилю. → /impeccable clarify

## Persona Red Flags

- **Alex**: длинный плоский список в «Другой шаблон»; Стандартный открывается «📋 Повесткой» перед «✅ Решениями» — решения не на первом экране; двухшаговый фидбэк.
- **Casey**: предупреждения качества отдельным сообщением после многочастного протокола — легко проскроллить; 3-частный протокол = 5+ сообщений подряд.
- **Руководитель с форвардом**: PDF пересылается без правок (имя по встрече, без бренда, без тофу). Риски: сводка-«мета-карточка», принятая за протокол; ℹ️-пометка о «Участник N» не уедет вместе с документом; в PDF две даты (генерации в колонтитуле и встречи в теле).

## Minor Observations

- QuickFeedbackManager.request_quick_feedback («Как вам протокол?») нигде не вызывается — мёртвая поверхность (для форвард-сценария отсутствие «оцените»-спама скорее плюс — решить судьбу).
- Дрейф ADR-0001: упоминает strip_heading_emoji, в коде strip_emoji.
- proto_regen_go: split("_") с распаковкой в 5 переменных — предполагает числовые id.
- Мёртвые подписи категорий product/sales без соответствующих шаблонов.
- Пустые {% if %}-секции в PDF дают несколько Spacer подряд — вертикальные дыры.
- Стаб _get_type_specific_formatting_instructions (deprecated, возвращает "") всё ещё в коде.

## Questions to Consider

1. Стандартный шаблон открывается «📋 Повесткой» перед «✅ Решениями» — принцип №5 обещает решения «глазами за секунды». Повестка важнее решений на первом экране?
2. Должен ли перегенерированный протокол гарантированно совпадать по именам участников с v1, если v1 уже ушёл наверх?
3. Читатель пересланных чат-частей не знает, когда протокол сгенерирован — только когда была встреча. Нужна ли дата создания вне PDF?
