# Устранение неполадок

## Частые проблемы

### 1. "File too large" (Файл слишком большой)
-   **Причина**: Telegram ограничивает ботов загрузкой файлов до 20MB.
-   **Решение**: Загрузите файл на Google Drive или Яндекс.Диск и отправьте ссылку боту.

### 2. "Transcription failed" (Ошибка транскрипции)
-   **Причина**: Аудио может быть повреждено, содержать тишину или быть в неподдерживаемом формате.
-   **Решение**: Проверьте файл локально. Если он воспроизводится, попробуйте конвертировать его в MP3/WAV перед загрузкой.

### 3. Ошибки OOM (Out of Memory / Нехватка памяти)
-   **Симптомы**: Бот падает или перезагружается при обработке больших файлов.
-   **Решение**:
    -   Увеличьте лимит памяти Docker.
    -   Переключите `TRANSCRIPTION_MODE` на `cloud` или `groq` для переноса нагрузки.
    -   Уменьшите `MAX_SPEAKERS` в конфигурации.

### 4. "LLM Error"
-   **Причина**: Проблемы с API ключами или сбои провайдера.
-   **Решение**: Проверьте ваши API ключи в `.env`. Проверьте баланс у провайдера (OpenAI, Anthropic и т.д.).

## Логи

Если вы хостите бота, проверьте логи для детальных сообщений об ошибках:

```bash
# Docker
docker-compose logs -f --tail=100

# Локально
tail -f logs/bot.log
```

## Получение помощи

Если вы столкнулись с багом, пожалуйста, откройте issue на GitHub с:
1.  Описанием проблемы.
2.  Шагами для воспроизведения.
3.  Соответствующими фрагментами логов.
