# Улучшение промптов для генерации протоколов

> **Статус:** Исследование завершено
> **Дата:** 2026-03-08
> **Цель:** Повысить качество генерации протоколов за счёт исправления багов, добавления недостающих правил и унификации контекста

---

## Контекст

Бот Soroka использует двухэтапный LLM-пайплайн:
- **Этап 1 (Анализ):** определение типа встречи + маппинг SPEAKER_N → имена
- **Этап 2 (Генерация):** извлечение полей протокола по переменным шаблона

Обнаружены три ключевые проблемы:
1. **Баг: Anthropic не передаёт повестку/проекты** в Этап 2 (строки 873-878 `llm_providers.py`)
2. **Баг: YandexGPT не поддерживает повестку/проекты** вообще (строки 976-985 `llm_providers.py`)
3. **~20 полей образовательных шаблонов без правил извлечения** — LLM не получает инструкций по ним

---

## Фаза 1: Исправление багов передачи контекста

### 1.1 Anthropic — добавить meeting_agenda и project_list в Этап 2

**Файл:** `llm_providers.py:873-878`

Сейчас:
```python
generation_user_prompt = build_generation_prompt(
    transcription=analysis_transcription,

```

Нужно добавить (как в OpenAI, строки 714-715):
```python
    meeting_agenda=kwargs.get('meeting_agenda'),
    project_list=kwargs.get('project_list')
```

### 1.2 YandexGPT — добавить meeting_agenda и project_list в _build_user_prompt

**Файл:** `llm_providers.py:352-603` (сигнатура `_build_user_prompt`) и `llm_providers.py:976-985` (вызов)

- Добавить параметры `meeting_agenda` и `project_list` в `_build_user_prompt()`
- Передать их при вызове из `YandexGPTProvider.generate_protocol()`
- Вставить блок контекста (повестка/проекты) в промпт, аналогично `build_generation_prompt()`

## Фаза 3: Улучшение системного промпта генерации

**Файл:** `src/prompts/prompts.py` — функция `build_generation_system_prompt()`

Текущий системный промпт (строки 740-754) слишком краток — 6 строк общих инструкций. Нужно расширить:

1. **Добавить инструкцию по использованию повестки/проектов** — если предоставлена повестка, структурировать протокол по её пунктам
2. **Добавить правила обработки длинных транскрипций** — не терять информацию из середины/конца
3. **Усилить инструкции по атрибуции** — формат цитирования, обработка групповых высказываний
4. **Добавить инструкцию о приоритете полей** — если информации много, discussion должен быть максимально полным
