# ADR-0005: Интерактивные экраны рендерятся в Telegram HTML

Дата: 2026-07-21. Статус: принято. Расширяет ADR-0001.

## Контекст

ADR-0001 перевёл тело Протокола на канальный рендер (канонический Markdown →
Telegram HTML), но интерактивные экраны — прежде всего Карточка сопоставления —
остались на MarkdownV2: ~18 спецсимволов ручного экранирования, тройной
fallback на одну отправку (~123 строки), параллельная «упрощённая» копия текста
(заголовки уже разошлись) и пять копий таблицы экранирования по репозиторию.
Тесты знали про экранированные строки (`SPEAKER\_1`).

## Решение

Интерактивные экраны отправляются через один шов — **отправитель карточек**:
модуль принимает семантическое содержимое (заголовок, строки, цитаты) и сам
отвечает за разметку **Telegram HTML** (экранирование только `&`, `<`, `>`),
единственную plain-страховку и редактирование/перерисовку. Один язык разметки
на весь бот; MarkdownV2-экранирование остаётся только в ещё не мигрированных
местах и уходит по мере касания.

## Последствия

- Смена формулировки или разметки карточки — одно место; копия-двойник текста
  удаляется.
- Тесты проверяют содержимое, а не экранирование.
- Пять копий `escape_markdown` схлопываются к канону по мере миграции экранов.
