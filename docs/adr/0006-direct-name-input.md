# ADR-0006: Прямой ввод имён в Карточке сопоставления

Дата: 2026-07-24. Статус: принято. Ревизует кнопочную часть ADR-0002.

## Контекст

По ADR-0002 назвать спикера человеком не из списка можно было так: в под-виде
выбора участника нажать «✏️ Ввести имя вручную» (callback `sm_custom`), попасть
на промежуточный экран «Отправьте имя для SPEAKER_N сообщением» и только затем
отправить имя. Три шага там, где просится один. Хуже того, в самом под-виде не
было видно, кого редактируешь: заголовок не называл спикера. Признак «пользователь
вводит имя» жил в отдельном реестре ожидания (#98) — временной конструкции рядом
с сессией сопоставления и её TTL.

## Решение

1. **Тап на спикера сразу открывает под-вид, готовый принять имя сообщением.**
   Кнопка спикера (`sm_change`) открывает под-вид, и отправленный текст
   становится именем этого спикера — без промежуточной кнопки и экрана. Кнопка
   «✏️ Ввести имя вручную», экран-приглашение и callback `sm_custom` упразднены.
   «❌ Оставить без имени», кнопки участников и «◀️ Назад» остаются.

2. **Заголовок под-вида называет спикера и приглашает печатать.** Без списка
   участников — «✏️ Имя для SPEAKER_N — отправьте сообщением»; со списком —
   «…отправьте сообщением или выберите ниже». Список спикеров с цитатами под
   заголовком остаётся: цитата напоминает, чей это голос.

3. **Признак «кто ждёт имя» — поле сессии `editing_speaker`.** Реестр ожидания
   из #98 удалён. Поле ставится при входе в под-вид (`sm_change`), снимается на
   «◀️ Назад», выборе участника, «Оставить без имени», успешном применении имени,
   подтверждении и пропуске. Ловец имени срабатывает, когда сессия жива И
   `editing_speaker` задан И content — текст.

4. **Фильтр пропуска: карточка не съедает то, что именем не является.** Ловец
   привязан только к тексту без ссылки. Текст со ссылкой (URL), файл, голосовое,
   видео, фото и прочий не-текст проваливаются мимо ловца в обычный поток
   обработки записи (общий `text_handler`); карточка остаётся открытой. «Ссылка»
   определяется тем же детектором, что и в общем обработчике записи, — чтобы то,
   что фильтр пропустил как ссылку, обработчик записи гарантированно опознал.

5. **Принцип: текст меняет карточку, необратимый шаг — только кнопка.** Ввод
   имени сообщением правит карточку на месте и обратим (можно переназвать или
   выбрать «Оставить без имени»). Продолжение обработки и пропуск — по-прежнему
   только кнопкой (`sm_confirm` / `sm_skip`): текст такой шаг не запускает.

## Решённые отказы

- **Планка имени 2–50 символов** после трима, не начинается с «/». Нижняя
  граница отсекает случайные «И», верхняя — вставленный абзац или ссылку,
  ошибочно принятые за имя. Вне планки — короткий отказ, ничего не применяем.
- **«Всё или ничего».** Невалидное имя не применяется даже частично: сопоставление
  и список участников не трогаются, под-вид продолжает ждать имя.
- **Одно имя в под-виде.** Несколько имён (разделители — запятая или перенос
  строки) → отказ с подсказкой: здесь ждём одно имя для SPEAKER_N, пачку имён
  можно отправить из общего вида карточки («◀️ Назад»).

## Считаемые последствия

- **Истёкшая по TTL сессия при открытом под-виде больше не отвечает «Состояние
  истекло».** Признак ожидания теперь живёт в самой сессии, поэтому нет живой
  сессии → нет под-вида → текст просто проваливается в общий обработчик
  («📎 Отправьте файл или ссылку…»). Прежний ответ «Состояние истекло» в этом
  крае исчезает — осознанный размен: под-вид без живой сессии невозможен.
- **Не-текст в под-виде больше не получает мягкий отказ «отправьте имя текстом».**
  Ловец привязан только к тексту (в отличие от нейтрального #98), файл/голос/фото
  уходят в обычную загрузку записи. Это осознанное изменение поведения.
- **Ссылка/файл, ушедшие мимо, не сбрасывают `editing_speaker`.** Под-вид
  продолжает ждать имя; пользователь выйдет «◀️ Назад» или тапнет кнопку.

## Отвергнутые альтернативы

- **Оставить промежуточную кнопку и экран (ADR-0002).** Три шага вместо одного и
  скрывают, кого редактируешь; ради этого держать отдельный реестр ожидания.
- **Ловить любой контент, а не только текст.** Тогда файл или ссылка, присланные
  при открытом под-виде, застряли бы в карточке вместо обработки как новая запись;
  фильтр пропуска для не-текста и ссылок обязателен.
- **Принимать несколько имён в под-виде.** Под-вид — про одного спикера; пачка
  имён — операция общего вида. Смешение усложнило бы и парс, и предсказуемость.
