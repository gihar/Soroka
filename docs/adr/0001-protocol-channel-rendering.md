# ADR-0001: Канонический Markdown протокола и канальный рендеринг

Дата: 2026-07-17. Статус: принято.

## Контекст

Шаблоны протоколов были написаны на GitHub-Markdown (`#`-заголовки, `**жирный**`),
а в чат протокол уходил с `parse_mode="Markdown"` (legacy), который эту разметку
не поддерживает: пользователи видели буквальные `##` и потерянный жирный.
Разбиение длинных протоколов резало текст по строкам без учёта секций.
UX-критика 2026-07-17 (19/40, снапшот в `.impeccable/critique/`) оценила это как P0.

У пользователей есть собственные Jinja-шаблоны — их нельзя молча сломать.

## Решение

1. **Канонический формат протокола — Markdown.** Его создают Jinja-шаблоны,
   системные и пользовательские; `ProcessingResult.protocol_text` остаётся
   Markdown-текстом. Кастомные шаблоны работают без миграции.
2. **Представление под канал — отдельный слой** `src/services/protocol_render/`:
   - чат Telegram: `markdown_to_telegram_html` (`#`/`**` → `<b>`, экранирование,
     `parse_mode="HTML"`) и `render_protocol_messages` — разбиение по границам
     `##`-секций, при разрыве секции её заголовок повторяется с «(продолжение)»;
   - файл `.md`: канонический текст без преобразований;
   - PDF: `src/utils/pdf_converter` стилизует тот же Markdown; эмодзи-метки
     секций снимаются (`strip_heading_emoji`) — в TTF-шрифтах нет их глифов.
3. **Единая структура системных шаблонов** (все семь живут в
   `TemplateLibrary`): шапка (`{{ meeting_title }}` с фолбэком, дата, участники) →
   ✅ Решения → 📌 Задачи и сроки → ⚠️ Блокеры и риски → спец-секции типа
   встречи → 💬 Обсуждение → 📅 следующие шаги. Каждая секция в `{% if %}`.
4. **Целевые поля вместо «резиновой» `discussion`**: `yesterday_progress`,
   `today_plans` (Дейли), `went_well`, `to_improve` (Ретро), `lecturer` (Лекция) —
   правила в `FIELD_SPECIFIC_RULES`.

## Последствия

- Читаемость в чате перестаёт зависеть от ограничений legacy-Markdown.
- Новые каналы (веб, e-mail) добавляются новым рендерером без правки шаблонов.
- Шаблон отвечает за структуру и контент; рендерер — за представление.
  Изменения вида в чате не требуют трогать шаблоны, и наоборот.
- Legacy-путь `parse_mode="Markdown"` для тела протокола удалён вместе со
  слепым сплитом `_split_protocol_text`.
