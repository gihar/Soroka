# Soroka

Telegram-бот, который превращает запись встречи в структурированный протокол: транскрибирует, разделяет по спикерам, сопоставляет спикеров с участниками и генерирует протокол по шаблону.

## Language

### Конвейер обработки

**Запись**:
Аудио/видео файл или внешняя ссылка, присланная пользователем на обработку.
_Avoid_: файл (слишком общо), медиа

**Встреча**:
Событие, зафиксированное записью; носитель темы, повестки, даты и участников.

**Транскрипция**:
Текст, полученный из записи распознаванием речи.
_Avoid_: расшифровка, стенограмма

**Диаризация**:
Разбиение транскрипции по анонимным голосам (Speaker N).

**Бэкенд транскрипции**:
Способ превратить запись в транскрипцию (whisper, groq, leopard, speechmatics, deepgram); выбирается режимом транскрипции, при сбое откатывается на whisper.
_Avoid_: провайдер транскрипции, движок

**Спикер**:
Анонимный голос из диаризации до сопоставления с участником.
_Avoid_: участник (до сопоставления это разные понятия)

**Участник**:
Человек на встрече — имя и роль, указанные пользователем или извлечённые из контекста.

**Сопоставление спикеров**:
Привязка анонимных спикеров к участникам, подтверждаемая пользователем.
_Avoid_: маппинг

**Сессия сопоставления**:
Приостановленная обработка, ожидающая подтверждения сопоставления пользователем; живёт до подтверждения, пропуска или истечения (1 час).
_Avoid_: состояние маппинга, mapping state

**Тип встречи**:
Классификация встречи (техническая, бизнес, обучающая, брейншторм, статус, общая); единственный источник — вердикт LLM (при сопоставлении спикеров или на этапе анализа генерации), влияет на промпт генерации.
_Avoid_: keyword-оценки подбора шаблона (это «категория шаблона», не тип встречи)

**Шаблон**:
Структура протокола с переменными; бывает системным или пользовательским.

**Протокол**:
Итоговый структурированный документ по встрече — продукт бота.
_Avoid_: отчёт, саммари

**Пресет модели**:
Именованная конфигурация LLM (модель, адрес, ключ); настоящая точка выбора «какая модель отвечает».
_Avoid_: LLM-провайдер (историческое имя того же понятия)

### Учёт и данные

**Пользователь**:
Владелец диалога с ботом; идентифицируется Telegram ID.

**Очередь обработки**:
Задачи, ожидающие обработки или находящиеся в работе.
_Avoid_: история (очередь — про незавершённое)

**История обработки**:
Свершившиеся результаты обработки записей; из неё считается статистика пользователя.
_Avoid_: очередь, результаты (по отдельности)

**Обратная связь**:
Оценки и комментарии пользователей о качестве протоколов.
_Avoid_: фидбек

**Метрики обработки**:
Технические измерения конвейера (длительности этапов, ошибки) для наблюдаемости.
_Avoid_: статистика (статистика — про пользователя, см. история обработки)

**Настройки приложения**:
Глобальные параметры бота, задаваемые администратором (например, активный пресет модели).
